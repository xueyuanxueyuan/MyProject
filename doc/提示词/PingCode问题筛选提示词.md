# PingCode 问题筛选提示词

## 1) 筛选我名下 `新提交` 和 `需修复` 的问题

请使用本地 Chrome + CDP 连接 PingCode，并严格按下面步骤执行，不要自行简化，也不要跳步：

1. 打开固定地址：  
   `https://pingcode.capinfo.com.cn/pjm/projects/GJJ-TSHS/fwwFWqFj/WlytPXXa`

2. 先检查是否已登录：
   - 如果页面标题是 `LDAP` 或出现登录页，暂停并提示我手动登录；
   - 登录完成后再继续。

3. 进入工作项列表后，先点击 `我负责的工作项`，不要直接在 `全部工作项` 里做状态筛选。

4. 再点击顶部 `筛选` 按钮，必须确认筛选面板真实打开。
   - 如果用户明确表示“没有看到筛选弹窗”，优先检查 Chrome 中是否存在多个相同的 PingCode 标签页。
   - 若存在多个相同标签页，必须切到每个命中的标签页分别点一次 `筛选`，直到用户确认当前看的那一页已经看到弹层。

5. 打开筛选面板后，先检查当前默认条件：
   - 正常应已有一条：`负责人 属于 当前用户`
   - 如果这条条件已存在，不要删除，不要改掉
   - 不要优先点 `重置`，除非筛选面板已经脏乱到无法识别当前条件
   - 在继续之前，最好读取 `.selected-value span`，确认已命中 `负责人`、`属于`、`当前用户`

6. 在现有 `负责人 属于 当前用户` 后，点击 `添加筛选条件` 新增第二条条件。

7. 第二条条件必须按下面顺序设置：
   - 属性：从 `标题` 改成 `状态`
   - 操作符：保持 `属于`
   - 值：勾选 `新提交` 和 `需修复`

7.1 修改第二条条件时，必须只操作“新增加的那一行”：
   - 不允许按“全局第几个下拉框”去点
   - 必须先确认第一行仍然是 `负责人 属于 当前用户`
   - 必须确认第二行还是默认的 `标题 包含`
   - 然后只把第二行从 `标题` 改成 `状态`
   - 如果改完后第一行变成了 `状态`，说明点错了行，必须立刻 `重置` 后重来
   - 当前项目页本次成功时，稳定读取到：
     - 属性控件：`['负责人', '', '标题', '']`
     - 值控件：`['当前用户', '', '选择状态', '']`
   - 也就是说，第二行属性通常对应第 `3` 个属性控件，第二行值通常对应第 `3` 个值控件

8. 选择状态值时，必须确认命中的是：
   - `新提交`
   - `需修复`
   不要误选 `已通过`、`已修复`、`打开`、`进行中` 等状态。

9. 设置完成后点击 `确定` 应用筛选。

9.1 如果基础筛选面板的第二行值选择器不稳定，优先切换到 `PQL` 模式作为兜底方案：
   - 已确认 PingCode PQL 支持 `currentUser()` 函数
   - 已确认支持 `in [A,B]` 语法
   - 推荐筛选语句：
     `负责人 = currentUser() and 状态 in [新提交,需修复]`
   - 输入后点击 `确定`，再按结果区做二次校验

10. 筛选完成后必须再次校验结果是否生效：
   - 顶部筛选徽标数量应为 `2`
   - 列表总数应明显收敛
   - 列表中的问题状态应只包含 `新提交` 或 `需修复`
   - 不应再出现 `已通过`、`已修复` 等状态

11. 最终返回结果时，输出：
   - 当前筛选条件
   - 当前命中的工作项数量
   - 前 5 条问题的编号、标题、状态

## 2) 执行经验

执行时务必遵守以下经验规则：

- 如果用户说“我没看到你打开筛选面板”，先排查是不是点到了另一个重复标签页，而不是立刻怀疑点击失败。
- 先点 `我负责的工作项`，再加状态筛选，这是这次成功的关键路径。
- 不要依赖模糊文本匹配直接判断筛选是否成功，必须读取页面结果二次校验。
- 如果 `添加筛选条件` 后默认出现的是 `标题 包含`，要显式把 `标题` 切换成 `状态`。
- 第二条条件必须按“按行定位”的方式处理，不能用全局索引，否则极容易把第一行 `负责人` 改坏。
- 如果第二行值控件点错，会弹出 `角色 / 成员 / 部门 / 团队`，这说明你点到的是第一行负责人值类型控件，不是第二行的 `选择状态`。
- 如果状态下拉已展开，要直接点击 `新提交` 和 `需修复` 两个选项，再点 `确定`。
- 如果页面显示仍是大量 `已通过` 数据，说明筛选未真正生效，需要重新检查是否仍停留在 `全部工作项` 或第二条条件没有改成 `状态`。
- 如果基础模式浮层抖动或下拉值没有真正提交，直接切换到 `PQL` 模式，使用：
  `负责人 = currentUser() and 状态 in [新提交,需修复]`

## 3) 成功判定标准

下次执行后，只要同时满足下面 4 条，就可以认为筛选成功：

- 顶部筛选徽标数为 `2`
- 工作项数量明显减少
- 列表状态只剩 `新提交` / `需修复`
- 返回结果能列出前 5 条命中问题
