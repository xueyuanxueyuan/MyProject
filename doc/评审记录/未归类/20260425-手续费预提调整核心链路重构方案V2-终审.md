# 手续费预提调整核心链路重构方案 V2.0 终审意见

## 1. 评审对象
- 文档：`doc/设计文档/手续费预提调整核心链路重构与优化方案_V2.0.md`
- 评审方式：终审，重点核对其与前两轮对抗性评审要求的一致性，以及是否达到可实施的 `95+` 水准

## 2. 终审结论
- 综合评分：**96 / 100**
- 结论：**可以批准实施**。
- 总体判断：方案主方向已经正确，前两轮评审提出的核心风险已基本被吸收，尤其是事务回滚、`null` 原值条件更新、数据库快照权威性、零差额不记账、正负号校验和批内整体回滚等关键点都已纳入正式设计与验收标准。

## 3. 主要问题项

### 3.1 文档版本标识存在轻微不一致
- 修订记录已经新增了 `V2.1.0`，并标注“批准实施 (96分)”。
- 但文档头部仍写着“当前版本：V2.0.0”，文件名也仍为 `..._V2.0.md`。
- 这不会影响技术落地，但会影响文档归档一致性与后续检索。
- 处理建议：
  - 头部版本号同步更新为 `V2.1.0`；
  - 如项目允许，文件名也同步调整为 `..._V2.1.md` 或在文档头部注明“文件名沿用历史命名”。

### 3.2 路线图中的测试项描述略滞后于最终验收口径
- `M3` 里程碑仍写的是“覆盖 Null ID、并发覆盖、零差额边界、RPC 异常四类核心用例”。
- 但最终验收标准已经扩展为：
  - 金额符号校验；
  - `NULL` 原值校验；
  - 批内回滚组合用例。
- 这是文档前后表述的一致性问题，不是方案方向问题。
- 处理建议：
  - 将 `M3` 的描述同步更新为“覆盖参数、防并发、零差额、符号方向、NULL 原值、批内回滚、RPC 异常等核心用例”。

## 4. 认可项
- 认可将 `mxMap` 明确为数据库权威快照来源，而非冗余结构。
- 认可“前置准备 -> 平衡校验 -> 集中落库”的三段式事务设计。
- 认可对 `rawSxf` 的 `isNull/eq` 条件更新语义修正。
- 认可“零差额不记账、明细保留符号、主表使用绝对值汇总”的业务口径表达。
- 认可将强门禁策略和业务确认一起写入风险应对部分。
- 认可在验收标准中新增“金额符号校验”“NULL 原值校验”“批内回滚组合用例”三项强卡点，这正是方案从 93 分提升到 96 分的关键。
