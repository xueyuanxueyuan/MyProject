## 4) Feign 降级治理（专项）
请治理当前模块 Feign 降级代码，重点处理 `return null`。

要求：
1. 不改接口签名。
2. 降级逻辑必须可观测：日志 + `R.fail(...)` / `BizException`。
3. 优先处理交易发起、结果查询、记账通知相关接口。
4. 输出修复清单与剩余风险清单。

## 5) 代码评审（项目规范视角）
请对本次改动做代码评审，重点关注：
1. 分层是否清晰，Controller 是否过重。
2. 是否存在空降级、吞异常、无日志失败分支。
3. 交易/审批状态流转是否完整且可追踪。
4. SQL 是否存在全表扫描风险。
5. 外部调用失败是否可观测且可恢复。

请按严重级别输出：`致命` / `高` / `中` / `建议`。

---

## 6) 提交前自检
请按以下清单检查当前变更并给出“通过/不通过”：
- 编译是否通过；
- 返回是否统一 `R<T>`；
- 校验、日志、注解是否齐全；
- Feign 降级是否无 `return null`；
- SQL 是否具备分页与合理筛选；
- 是否包含最小回归验证步骤。
