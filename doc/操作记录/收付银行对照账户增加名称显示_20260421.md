# 收付银行对照账号增加名称显示 操作记录

## 需求背景
在收付银行对照功能中的“银行账户号码”选择器里，需要在显示账号（`yhzhhm`）的后面同时显示账户名称（`yhzhmc`），以方便用户更直观地选择银行账户。

## 分析思路
1. **分析数据结构**：
   - 银行账户下拉列表的数据由 `yhzhxxList` 接口提供。
   - 分析对应的 Java 实体和返回的 DTO 数据结构可知，`yhzhmc` 即为“银行专户名称”。
2. **定位相关前端代码**：
   - `src/components/cwhs/sfyhdz.vue` 中获取银行账户信息的函数 `getYhzhxx` 循环组装了 `new_item` 对象并 push 到 `yhzhxxs` 中。
   - 原代码中，`new_item` 只包含了 `id`, `zhjc`, `yhzhhm`, `yhdm` 等字段。
   - 页面上有单选 (`prop="zhxxId"`) 和多选 (`prop="zhxxIds"`) 两个 el-select 控件使用 `yhzhxxs` 数据源。

## 解决步骤
1. **完善数据源**：
   在 `sfyhdz.vue` 的 `getYhzhxx` 方法中，在构造 `new_item` 对象时，补充添加 `yhzhmc: item.yhzhmc` 属性：
   ```javascript
   let new_item = {
       id: item.id,
       zhjc: item.zhjc + "（" + item.yhdmmc + "）",
       yhzhhm: item.yhzhhm,
       yhzhmc: item.yhzhmc,
       yhdm: item.yhdm
   }
   ```
2. **修改显示模板**：
   在单选和多选的 `<el-option>` 组件中，将原本仅绑定 `item.yhzhhm` 的 `:label` 属性更新为包含账户名称的形式：
   ```vue
   <el-option v-for="item in yhzhxxs"
              :key="item.id"
              :label="`${item.yhzhhm}（${item.yhzhmc}）`"
              :value="item.id">
   </el-option>
   ```

## 验证与留痕
- 通过本次修改，银行专户选择器的下拉选项及回显值都能显示“账号（账户名称）”的格式。
