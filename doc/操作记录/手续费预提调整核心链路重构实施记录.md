# 手续费预提调整核心链路重构实施记录

## 1. 实施背景
根据《手续费预提调整核心链路重构与优化方案_V2.1.md》的设计规范，对 `CwhsServiceImpl.java` 中的 `batchSaveSxfyttz` 方法进行系统级重构，并抽离出私有的 `generateSxfyttzPz` 记账凭证生成方法。

## 2. 实施步骤
### 2.1 方法签名与前置防御
- **防御性编程拦截**：在方法最开始增加 `CollUtil.isEmpty(req.getItemList())` 的校验。
- **状态强门禁**：通过 `zbkhFeignClient.queryJdhzhList` 查询排名更新状态，若状态 `>= "03"` 或接口调用异常时，直接返回失败（强门禁原则）。
- **去重与校验**：对前端传入的 `mxId` 进行 `distinct` 过滤并查验数量是否匹配。

### 2.2 权威数据快照获取
- 提取明细ID后，调用 `sxfytMxDmService.listByIds` 获取底层数据库数据，并转为 `mxMap` 作为权威数据源，丢弃前端传入的维度数据（防篡改）。

### 2.3 内存计算与条件更新准备
- 遍历传入的项目：
  - 判断底层快照金额 `currentSxf` 与前端声明的调整前金额 `ytqje` 是否一致。
  - 计算本次调整差额 `yttzce = ythje - currentSxf`。
  - 构造 `LambdaUpdateWrapper`，在 SQL 层利用 `isNull`（原值为 null）或 `eq`（原值非 null）对 `sxf` 字段进行精确匹配，彻底杜绝“丢失更新”的并发安全隐患。
  - 将差额不为0（`yttzce != 0`）的数据收集到 `tzItemList` 以供后续记账使用。

### 2.4 集中落库与事务保护
- 在更新前校验调整前后的总额是否平衡。
- 采用集中循环调用 `update` 方法，任意一条更新失败则立刻 `throw new BizException(...)`，触发 `@Transactional` 完整回滚，消除脏数据隐患。

### 2.5 凭证生成抽离
- 编写 `generateSxfyttzPz` 方法处理记账逻辑：
  - 依据收费类别判定凭证的业务类型（归集/个贷）。
  - 根据数据库权威数据组装 `CwjzYwlsMxDTO` 列表，明细记账金额保留符号（正负差额透传）。
  - 计算主表凭证的总业务发生额（明细绝对值之和除以2）。
  - 调用 `cwhsService.receiveCwjzxx` 进行财务系统入账对接。

## 3. 验证情况
- 依赖 `GetDiagnostics` 执行静态语法检查，未发现任何编译级别错误。
- 逻辑严格匹配 V2.1.0 验收标准的五大核心点：基础防御、零差额过滤、金额符号校验、NULL原值校验、组合回滚。
