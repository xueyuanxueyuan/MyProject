# 手续费预提调整凭证生成逻辑及实现

## 1. 业务背景
在手续费预提调整（`sxfyttz`）模块中，用户能够在预提后、考核汇总前，针对特定网点调整预提金额。调整完成后，不仅要保存金额变化，还需要在财务核算系统中自动生成对应的财务记账凭证。

之前尝试将生成凭证作为独立按钮，但为了提升操作连贯性与系统自动化程度，现**将财务记账逻辑集成至“保存预提调整”逻辑内部**，并且**业务类型沿用原本的“手续费预提”**。

## 2. 逻辑分析

### 2.1 前端逻辑
- 页面路径：`sxfyttz.vue`
- 原有逻辑：用户通过页面输入调整金额（`ythje`），前端将调整数据传递至后端的 `batchSaveSxfyttz` 进行保存。
- 本次改动：移除独立生成凭证按钮。由原来的独立触发，变为保存后自动触发。

### 2.2 后端逻辑
- 核心服务层实现 (`CwhsServiceImpl.batchSaveSxfyttz`)：
  1. 接收前端批量调整请求，逐条更新 `SxfytMxDO` 的预提金额与调整差额。
  2. 更新完毕后，根据内存中计算出的差额（`ythje - ytqje`），过滤出有实际差额（`yttzce != 0`）的数据集合。
  3. **业务类型**：不再新建独立的业务类型枚举，而是根据当前调整数据所归属的考核类型，复用对应的 `YWLX_SXFYT_GD`（个贷手续费预提）或 `YWLX_SXFYT_GJ`（归集手续费预提）。
  4. **金额记账规则**：
     - 只针对差额（调增调减部分）作记账。
     - 调增部分，金额为正数。
     - 调减部分，金额为负数。
     - 在组装 `CwjzYwlsMxDTO`（明细）时，统一将带符号的差额放入增加额（`zje`），而减少额（`jse`）记为 `0`。
     - 主表 `CwjzYwlsDTO` 中的业务发生额（`ywfse`），则取所有差额的绝对值之和的一半，来满足单边金额的记账记录要求。
  5. 调用已有的公共财务记账方法 `receiveCwjzxx` 进行凭证自动生成及持久化。

## 3. 实现步骤记录

1. **前端修改**：
   - 移除 `sxfyttz.vue` 中多余的 `生成凭证` 按钮及对应的 `handleScpz` 方法。

2. **后端清理**：
   - 移除之前临时添加的 `SxfyttzPzScReqDTO` 和对应的 Controller、Service 方法。
   - 移除临时增加的业务类型枚举 `YWLX_SXFYT_TZ`。

3. **后端核心记账逻辑集成**：
   - 在 `CwhsServiceImpl` 的 `batchSaveSxfyttz` 方法末尾集成凭证生成代码：
     - 根据考核类型转换为原业务类型代码。
     - 根据正负差额统一使用 `zje` 赋值，实现“调增为正数，调减为负数”的直观记账模式。
     - 调用 `receiveCwjzxx` 自动完成凭证生成及核算落库。

## 4. 总结
通过本次调整，将财务凭证的生成由人工触发变为了业务动作的自动联动，大幅简化了前台操作流程；并且严格遵循了原有预提凭证业务类型以及正负金额记账逻辑，保障了财务数据连贯性与规范性。
