# Doc 目录子文件夹中文化及依赖同步记录

## 1. 背景与目标
用户提出将 `doc` 目录下的所有英文文件夹修改为中文命名，并且需要全局同步更新所有引用了这些目录的规则、技能及记忆，确保修改后平台（尤其是“三省六部”体系和各类 AI 技能）依然能准确找到对应文件。

## 2. 重命名映射关系
经过梳理，对 `doc` 下的文件夹制定了如下中英文映射并执行了 `git mv`：
- `doc/skills` ➔ `doc/技能库`
- `doc/rules` ➔ `doc/项目规范`
- `doc/prompt` ➔ `doc/提示词`
- `doc/records` ➔ `doc/操作记录`
- `doc/architecture` ➔ `doc/架构设计`
- `doc/analysis` ➔ `doc/需求分析`
- `doc/audit_reports` ➔ `doc/审计报告`
- `doc/demand` ➔ `doc/需求文档`
- `doc/design` ➔ `doc/设计文档`
- `doc/review` ➔ `doc/评审记录`

## 3. 全局依赖同步步骤
为保证平台可用性，对全局依赖进行了如下深度替换：
1. **[吏部] 全局记忆更新**：
   - 更新了 `文档与技能同步全局默认规范` 记忆中的路径（`doc/项目规范`、`doc/技能库` 等）。
   - 更新了 `礼部文档产出路径与临时文件规范` 中的路径（`doc/操作记录`、`doc/架构设计`）。
   - 更新了 `项目审计与开发者响应规范` 中的路径（`doc/审计报告`）。
2. **[工部] 代码库全局替换**：
   - 遍历了 `.trae`、`doc`、`edict-platform` 目录下的所有 Markdown 文件。
   - 使用 `sed` 命令批量将所有旧的英文路径（如 `doc/skills`）替换为了中文路径（如 `doc/技能库`）。
   - 重点更新了 `.trae/rules/edict-workflow.md`，保障“门下省”和“礼部”的工作流能够精准识别新目录。
3. **[兵部] 物理目录重命名**：
   - 最终执行了安全的 `git mv` 及 `mv`，确保重命名被 Git 追踪且不丢失历史。

## 4. 结论
目录结构已全面中文化，且所有智能体规则、技能定义和核心记忆均已无缝衔接至新目录。未来“礼部”自动生成的记录文件将直接存入 `doc/操作记录` 等对应中文目录中。
