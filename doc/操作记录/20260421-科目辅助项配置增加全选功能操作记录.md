# 科目辅助项配置（kmfzx.vue）增加全选功能操作记录

## 需求背景
根据用户要求，需要在科目辅助项配置页面（`kmfzx.vue`）的 `el-tabs` 内容区域中的 `el-tree` 上方，增加“全选”功能，以方便用户一键勾选或取消勾选该分类下的所有树节点。

## 分析思路
1. **组件定位**：
   - 用户在浏览器选中的区域为一个 `el-tab-pane`，内部包含一个 `el-tree`。
   - 通过代码检索，确认相关业务文件为 `zaozhuang/WebstormProjects/capinfo-gjj-frontend-jshs-gm/src/components/cwhs/kmfzxpz/kmfzx.vue`。
2. **逻辑设计**：
   - 在 `v-for="(item, index) in editableTabs"` 循环生成的每个 `el-tab-pane` 中，在 `el-tree` 上方增加一个 `<el-checkbox>` 用于全选。
   - 在 `editableTabs` 动态 push 时，为每个 tab 数据对象初始化 `checkAll: false` 和 `isIndeterminate: false` 属性。
   - 绑定 `@change` 事件 `handleCheckAllChange`：当复选框状态改变时，通过 `$refs['tree-'+item.name]` 调用 `setCheckedNodes(item.data)` 或 `setCheckedKeys([])` 实现全选和全不选。
   - 绑定 `el-tree` 的 `@check` 事件 `handleTreeCheck`：当用户在树节点上手动勾选时，递归计算整棵树的所有节点总数，并对比当前已勾选节点数量，实时更新该 tab 对应的 `checkAll`（全选）和 `isIndeterminate`（半选）状态。
   - 在新增 Tab 页面 (`addTab`) 时，通过 `$nextTick` 触发一次树节点状态检查，确保弹窗打开时或初始状态下，如果默认全选/半选，Checkbox 能够正确回显。

## 解决步骤
1. **修改 Template 结构**：
   - 在 `el-tab-pane` 内部增加了全选的 Checkbox。
   - 为 `el-tree` 绑定了 `@check` 回调事件。
2. **修改 Script 逻辑**：
   - `addTab` 方法内为推送的 `item` 新增 `checkAll` 和 `isIndeterminate` 字段，并通过 `$nextTick` 初始化状态。
   - 新增 `handleCheckAllChange(val, item)` 方法处理全选框改变事件。
   - 新增 `handleTreeCheck(item)` 方法，在树节点改变时计算所有节点数量并刷新对应的全选和半选状态。

## 验证与留痕
- “全选”按钮的状态与内部 `el-tree` 的状态完全解耦并相互联动，符合预期设计。
