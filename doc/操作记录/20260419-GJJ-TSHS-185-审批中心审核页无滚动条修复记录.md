# GJJ-TSHS-185 审批中心审核页无滚动条修复记录

## 问题描述

PingCode 问题：`GJJ-TSHS-185`

问题标题：

- `结算核算-审批中心，审核页面，信息项较多时，页面无滚动条，只能缩放比例查看，需优化`

## 定位结论

本问题不是单个详情组件字段太多导致的，而是审批中心公共审核弹窗容器的滚动布局问题。

对应页面：

- `src/components/spzx/jzsp.vue`

关键结构：

- 审批中心通过 `x-dialog` 打开全屏弹窗
- 弹窗中使用 `el-tabs` 承载：
  - `申请数据`
  - `申请材料`
  - `审批过程`

## 根因分析

当前样式中：

- `el-dialog__body` 虽然全局有 `overflow: auto`
- 但 `jzsp.vue` 中的 `el-tabs` 被固定成整块高度
- `el-tabs__content` 没有承担滚动职责

这会导致：

- 当 `申请数据` 页签中的详情组件字段较多时
- 内容被限制在 tabs 内容区内
- 页面不会出现可用的纵向滚动条
- 用户只能通过缩放浏览器比例勉强查看完整内容

## 修复方案

在 `src/components/spzx/jzsp.vue` 中对审批中心全屏页签容器做最小修复：

1. 将 `el-tabs` 改为纵向 `flex` 布局
2. 让 `el-tabs__content` 作为可滚动区域
3. 保持各详情组件不改动，只修公共容器

本次新增样式要点：

- `display: flex`
- `flex-direction: column`
- `el-tabs__content { flex: 1; min-height: 0; overflow-y: auto; }`

## 修复影响范围

本次修复作用于审批中心公共审核页：

- 所有通过 `spzx/jzsp.vue` 在“申请数据”页签中展示的详情组件

因此是一次公共收益修复，而不是只修某一个业务详情页。

## 风险评估

风险较低，原因如下：

- 仅调整审批中心容器滚动样式
- 不修改业务字段、不改接口、不改审批逻辑
- 不影响具体详情组件数据绑定

需要关注：

- `申请材料` / `审批过程` 页签在大屏和小屏下的滚动体验
- 左右分栏场景是否仍按原预期展示

## 验证建议

建议重点验证以下场景：

1. 打开审批中心任一字段较多的审核任务
2. 进入 `申请数据` 页签
3. 确认页面出现纵向滚动条
4. 不缩放浏览器比例时，可直接滚动查看完整字段
5. 切换到 `申请材料`、`审批过程` 页签，确认无明显布局回归
