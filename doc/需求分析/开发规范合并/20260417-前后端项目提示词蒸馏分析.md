# 前后端项目提示词蒸馏分析

## 1. 目标
- 将原本偏前端的项目提示词扩展为适用于当前 `Gjj` 工作区前后端项目的通用提示词。
- 在提示词中加入项目结构分析和新增代码放置约束，避免代码随意乱放。

## 2. 结构分析结果
### 2.1 后端
- 工作区内后端核心工程位于 `zaozhuang/IdeaProjects/capinfo-gjj-busi-jshs`。
- 后端采用 `Maven` 多模块聚合结构，聚合根下按业务域拆分多个子模块。
- 单业务模块通常进一步拆为：
  - `*-basic-svc-api`
  - `*-basic-svc-api-v2`
  - `*-basic-svc-app`
  - `*-basic-svc-busi`
- 实际代码样本显示：
  - `api` 层承载 DTO、Feign、常量、接口契约；
  - `app` 层承载启动类与配置；
  - `busi` 层承载 Controller、Service、Domain、DAO、Mapper、JobHandler 等业务实现。

### 2.2 前端
- 前端工程位于 `zaozhuang/WebstormProjects/capinfo-gjj-frontend-jshs-gm`。
- 前端结构清晰，已形成：
  - `src/api`
  - `src/components/common`
  - `src/components/<业务域>`
  - `src/common`
  - `src/router`
  - `src/store`
  - `src/assets`

## 3. 蒸馏后的核心约束
- 先识别项目、模块、层级，再编码。
- 新增代码必须放在正确的业务模块和职责层级。
- 单业务代码优先放本业务模块，不允许为了方便乱放进公共层。
- 只有真实复用的能力才能上升到公共层或核心模块。
- 重构时必须保证原业务逻辑不变，且不能影响其他业务。
- 开发完成后必须自检，确认业务逻辑完整可用。
