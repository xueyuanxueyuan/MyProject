# 项目提示词蒸馏分析

## 1. 目标
- 基于 `capinfo-gjj-frontend-jshs-gm` 项目现状，蒸馏一份可直接用于智能体协作的前端开发提示词。

## 2. 提炼依据
- `doc/项目规范/Gjj项目组开发规范.md`
- `doc/项目规范/Gjj项目组智能体协同开发规范.md`
- `doc/项目规范/项目开发规范.md`
- `doc/提示词/Gjj项目组开发提示词.md`

## 3. 项目关键特征
- 技术栈固定为 `Vue 2 + Vue CLI + Element UI + Vuex + Vue Router + Axios`。
- 页面开发高度依赖现有公共能力，尤其是 `x-table`、`x-dialog`、`cap-form`、字典与权限能力。
- 项目强调配置驱动式页面开发，不适合使用脱离现状的通用模板。
- 重构与公共改动必须保证不影响原业务逻辑及其他业务页面。
- 交付前必须完成自检并说明验证范围。

## 4. 蒸馏策略
- 保留一份“通用主提示词”，用于绝大多数前端开发任务。
- 增补“新增功能、缺陷修复、重构优化、代码评审、提交前自检”五类专项提示词，便于直接复制使用。
- 将项目约束前置写入提示词，减少智能体偏离项目真实架构和风格的概率。

## 5. 产出结果
- 新增提示词文档：`doc/提示词/capinfo-gjj-frontend-jshs-gm前端开发提示词.md`
