# 枣庄市个贷资金使用管理-资金调拨需求分析-最终版

> 版本日期：2026-03-31  
> 依据评审：`20260331-资金调拨评审意见-v1.5-最终确定版-评审记录.md`

## 1. 分析范围

| 模块 | 范围说明 |
| --- | --- |
| 1.1.1.2 | 贷款发放资金确认中的“资金调拨申请”能力 |
| 1.1.1.3 | 贷款资金调拨全流程管理（审批、支付、状态流转） |

## 2. 主业务链路（最终口径）

| 步骤 | 触发条件 | 系统动作 | 结果 |
| --- | --- | --- | --- |
| 1 | 按银行汇总待放款 | 计算可用放款金额=`当前账户余额-账户备付金` | 得到可放款能力 |
| 2 | 可用金额>=待放款汇总 | 执行放款确认 | 全量放款完成 |
| 3 | 可用金额<待放款汇总 | 先按提交时间顺序确认可满足部分 | 形成未放款缺口 |
| 4 | 存在未放款缺口 | 发起集中资金调拨申请并附未放款清册 | 进入调拨审批流程 |
| 5 | 审批通过 | 发起调拨支付 | 进入支付与结算流程 |

## 3. A 清单：硬需求与底线约束（最终生效）

| 编号 | 规则 | 归属 | 验收属性 |
| --- | --- | --- | --- |
| A-001 | 可用放款金额小于 0 展示为 0 | 原需求硬约束 | 阻断 |
| A-002 | 放款确认仅在可用放款金额不等于 0 可点击 | 原需求硬约束 | 阻断 |
| A-003 | 余额不足时，未放款部分发起调拨并附未放款清册 | 原需求硬约束 | 阻断 |
| A-004 | 调拨申请采用余额不足账户集中发起 | 原需求硬约束 | 阻断 |
| A-005 | 调拨支付字段固定：收/付款行、收/付款账号、金额、状态 | 原需求硬约束 | 阻断 |
| A-006 | 取消/作废操作必须在同一事务内释放占用额度与缺口单据锁定 | 评审升级硬约束 | 阻断 |
| A-007 | 调拨金额必须 `>0`，精度 2 位小数，计算与存储统一 `BigDecimal` | 评审升级硬约束 | 阻断 |
| A-008 | 单据达到终态时，若存在部分失败明细，必须释放失败明细关联的缺口并允许重新发起新单 | 评审升级硬约束 | 阻断 |
| A-009 | 调拨支付执行前必须实时校验付款账户当前可用余额，余额不足强阻断 | 评审升级硬约束 | 阻断 |
| A-010 | 调拨支付逐条幂等标识与防重放必须实现 | 评审升级硬约束 | 阻断 |
| A-011 | 业务层禁止手动重试，但针对网络超时等链路抖动，技术层必须实现同幂等键的自动重投机制 | 评审升级硬约束 | 阻断 |
| A-012 | 数据库模型必须将状态拆分为双字段：`办结状态` 与 `结果状态(全成功/部分成功/全失败)` | 评审升级硬约束 | 阻断 |
| A-013 | 金额必须满足三层校验：单行<=缺口，且同付款账号汇总<=实时可用，且整单汇总<=缺口总额 | 评审升级硬约束 | 阻断 |

## 5. 关键差异落地表（相对旧口径）

| 差异点 | 旧口径 | 最终版口径 | 处理结论 | 状态 |
| --- | --- | --- | --- | --- |
| 幂等与防重 | D 清单增强项 | 升级为 A 清单硬需求 | 提测阻断项 | 生效 |
| 金额校验 | 待产品确认 | 行级、同付款账号汇总级、整单汇总级**三层拦截**；后端强制 `>0`、2位小数 | 技术底线固化，防并发拆单透支 | 生效 |
| 失败处理策略 | 明细级重试/待确认 | 业务层原单直接办结重发新单；技术层针对抖动执行同幂等键**自动重投** | 兼顾系统极简与运营体验 | 生效 |
| 状态数据模型 | 单状态展示 | 前端展示聚合，后端数据库强制拆分**双字段**（办结状态+结果状态） | 满足审计与复杂报表统计要求 | 生效 |
| 取消后行为 | 仅作废语义 | 作废同时事务释放占用资源 | 防资金池死锁 | 生效 |
| 附件落点 | 建议补充入口 | 强制复用全局附件组件 | 统一实现边界 | 生效 |

## 6. 字段与校验最终口径（核心）

| 页面 | 字段 | 默认值 | 校验规则 | 备注 |
| --- | --- | --- | --- | --- |
| 资金确认列表 | 可用放款金额 | 计算值 | `<0` 显示 `0` | 只读 |
| 资金确认列表 | 放款确认按钮 | 无 | 仅 `可用放款金额<>0` 可用 | 操作约束 |
| 调拨清册 | 调拨付款账号 | 可用余额最大银行 | 下拉可改 | 反显付款银行 |
| 调拨清册 | 调拨金额 | 缺口资金 | 必填、`0 < 调拨金额 <= 缺口资金` | 精度 2 位小数，`BigDecimal` |

## 7. 测试验收基线（阻断项）

| 验收项 | 最低通过标准 | 阻断级别 |
| --- | --- | --- |
| 幂等防重 | 同一明细重复请求不得重复打款 | 阻断 |
| 余额前置拦截 | 支付前账户余额不足时必须拦截报错 | 阻断 |
| 金额防超额 | 篡改金额超过缺口资金/最大可用时必须拦截 | 阻断 |
| 资源释放(作废) | 申请作废后，相关占用资源必须可再次进入调拨池 | 阻断 |
| 资源释放(失败) | 单据终态(部分成功/全失败)后，失败明细关联的缺口必须释放，允许重新发起新业务单 | 阻断 |

## 8. 术语统一（执行）

| 标准术语 | 禁用别名 | 说明 |
| --- | --- | --- |
| 放款确认 | 发放确认 | 统一按钮、接口、测试命名 |
| 调拨支付 | 调拨执行 | 统一支付动作术语 |
| 调拨完成(含部分成功) | 部分失败/处理异常 | 指单据内所有明细已取得最终结果(无论成败)，原业务单据办结 |
| 重新发起 | 重新调拨/整单重发/明细重试 | 针对办结单据中失败释放的缺口，作为一笔全新的申请业务发起 |
