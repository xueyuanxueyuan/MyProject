## 1. 背景与目标

| 项目 | 内容 |
| --- | --- |
| 业务背景 | 贷款发放资金确认阶段会出现“账户可用资金不足”场景，需要将不足部分通过资金调拨完成闭环 |
| 当前定版目标 | 在满足原需求的前提下，固化资金安全底线、审计口径与可执行验收标准 |
| 定版结论 | 评审意见已达成一致，当前文档作为详细设计、开发、测试统一输入 |

## 3. 业务主链路（定版）

| 步骤 | 触发条件 | 处理动作 | 输出结果 |
| --- | --- | --- | --- |
| L1 | 按银行汇总待发放数据 | 计算可用放款金额=`当前账户余额-账户备付金` | 可用放款金额 |
| L2 | 可用放款金额>=待发放汇总金额 | 直接执行放款确认 | 全量放款完成 |
| L3 | 可用放款金额<待发放汇总金额 | 按提交时间顺序先放可满足部分 | 形成未放款缺口 |
| L4 | 存在缺口 | 发起集中调拨申请并附未放款清册 | 进入审批流 |
| L5 | 审批通过 | 发起调拨支付（支付前实时余额校验） | 进入结算流程 |
| L6 | 结算返回 | 单据办结并按结果状态归类（全成功/部分成功/全失败） | 进入终态 |

## 4. A 清单（硬约束，阻断级）

| 编号 | 硬约束 | 验收属性 |
| --- | --- | --- |
| A-001 | 可用放款金额小于0按0展示 | 阻断 |
| A-002 | 放款确认仅在可用放款金额不等于0可点击 | 阻断 |
| A-003 | 余额不足时，未放款部分必须发起调拨并附未放款清册 | 阻断 |
| A-004 | 调拨申请按余额不足账户集中发起 | 阻断 |
| A-005 | 调拨支付展示字段固定：收/付款行、收/付款账号、金额、状态 | 阻断 |
| A-006 | 取消/作废必须在同一事务内释放占用额度与缺口锁定 | 阻断 |
| A-007 | 调拨金额必须`>0`，精度2位小数，计算与存储使用`BigDecimal` | 阻断 |
| A-008 | 单据终态若存在失败明细，必须释放失败明细缺口并允许重新发起新单 | 阻断 |
| A-009 | 调拨支付执行前必须实时校验付款账户可用余额，余额不足强阻断 | 阻断 |
| A-010 | 调拨支付逐条幂等标识与防重放必须实现 | 阻断 |
| A-011 | 业务层禁止手动重试；技术层必须支持同幂等键自动重投（仅限同次执行窗口） | 阻断 |
| A-012 | 数据模型必须拆分双状态字段：办结状态、结果状态（全成功/部分成功/全失败） | 阻断 |
| A-013 | 金额必须三层校验：单行<=缺口、同付款账号汇总<=实时可用、整单汇总<=缺口总额 | 阻断 |

## 5. 状态模型（双字段）

| 字段 | 可选值 | 含义 | 用途 |
| --- | --- | --- | --- |
| 办结状态 `orderCloseStatus` | 未办结、已办结 | 是否结束当前业务单生命周期 | 控制是否还能在原单继续操作 |
| 结果状态 `orderResultStatus` | 全成功、部分成功、全失败 | 单据内明细执行结果 | 报表、审计、复盘统计 |

| 申请 | 是 | 是 | 是 | 否 | 是 |
| 审批中 | 否 | 否 | 否 | 否 | 是 |

## 7. 失败处理与重发策略（定版）

| 议题 | 定版规则 | 禁止项 | 业务效果 |
| --- | --- | --- | --- |
| 业务失败处理 | 原业务单据到终态后办结，失败明细释放缺口并走“新单重发” | 在原单据中人工明细重试 | 保持单据单向流转，降低复杂度 |
| 技术故障处理 | 同幂等键自动重投（同执行窗口），超限后转人工新单 | 使用新幂等键重复发起同一明细 | 降低链路抖动导致的人工补单量 |
| 状态回写 | 办结状态与结果状态同步回写 | 仅维护单一混合状态 | 支撑审计与统计 |

## 8. 字段与校验口径（核心页面）

| 页面 | 字段 | 默认值 | 校验规则 | 失败提示口径 |
| --- | --- | --- | --- | --- |
| 资金确认列表 | 可用放款金额 | 计算值 | `<0`显示`0` | 可用金额不足 |
| 资金确认列表 | 放款确认按钮 | 无 | 仅`可用放款金额<>0`可用 | 当前不可放款 |
| 调拨清册 | 调拨付款账号 | 可用余额最大银行 | 下拉可改并联动反显付款银行 | 付款账号无效 |
| 调拨清册 | 调拨金额 | 缺口资金 | `0<金额<=缺口`，2位小数，`BigDecimal` | 金额超限/格式错误 |
| 调拨清册 | 同付款账号汇总 | 计算值 | 汇总`<=`支付前实时可用 | 付款账号余额不足 |
| 调拨支付 | 支付前校验 | 无 | 实时余额校验、幂等校验、防重放校验 | 支付被拦截，请刷新后重试 |

## 9. 资金安全控制点

| 控制点 | 控制方式 | 触发时机 | 责任层 |
| --- | --- | --- | --- |
| 透支防控 | 支付前实时余额校验 | 点击调拨支付前 | 应用层+基础设施层 |
| 重复支付防控 | 明细级幂等键、防重放 | 每次支付请求 | 基础设施层 |
| 超额防控 | 三层金额校验 | 申请提交+支付执行 | 应用层 |
| 资源死锁防控 | 取消/作废事务内释放资源 | 状态变更到终态时 | 应用层+领域层 |

## 10. 验收基线（阻断项）

| 验收项 | 最低通过标准 | 阻断级别 |
| --- | --- | --- |
| 幂等防重 | 同一明细重复请求不得重复打款 | 阻断 |
| 余额前置拦截 | 支付前余额不足必须拦截并报错 | 阻断 |
| 金额防超额 | 任一层级超限均必须拦截 | 阻断 |
| 资源释放(作废) | 作废后资源可再次进入调拨池 | 阻断 |
| 资源释放(失败) | 终态后失败明细缺口已释放且可新单重发 | 阻断 |
| 双状态一致性 | 办结状态与结果状态组合合法且可追溯 | 阻断 |
| 自动重投边界 | 仅同幂等键同窗口自动重投，超限转人工新单 | 阻断 |

| 标准术语 | 禁用别名 | 说明 |
| --- | --- | --- |
| 放款确认 | 发放确认 | 统一按钮、接口、测试命名 |
