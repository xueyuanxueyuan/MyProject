# 枣庄公积金测试系统登录技能

本技能用于自动化登录枣庄公积金测试系统 (`http://test.zzgjj:9090/#/`)，通过 `browser-use` 技能实现浏览器操作。

## 适用场景
- 用户要求“登录枣庄测试系统”。
- 用户需要进行“系统验证”、“测试环境登录”。
- 登录后如果需要验证功能，将默认进入“结算核算子系统”。

## 登录账号信息
- **URL**: `http://test.zzgjj:9090/#/`
- **登录方式**: 账号登录
- **用户名**: `bjzw`
- **密码**: `123456`
- **验证码**: **交由用户手动输入**

## 执行步骤
1. 使用 `browser-use --connect open http://test.zzgjj:9090/#/` 连接本地Chrome并打开登录页面。
2. 使用 `browser-use state` 获取页面元素，找到“账号登录”的切换按钮并点击（如果默认不是账号登录）。
3. 找到用户名输入框，使用 `browser-use input <index> "bjzw"` 输入账号。
4. 找到密码输入框，使用 `browser-use input <index> "123456"` 输入密码。
5. **重要：输入完账号和密码后，必须停止自动化操作，并通过自然语言提示用户：“账号和密码已输入，请您手动在浏览器中输入验证码并点击登录”。**
6. 严禁尝试自动识别或自动输入验证码。
7. 当用户确认已手动输入验证码完成登录后，使用 `browser-use --connect state` 查看主工作台页面。
8. 找到“结算核算子系统”卡片中的“进入系统”按钮，使用 `browser-use click <index>` 点击进入。
9. 如遇新窗口或新标签页，使用 `browser-use switch <tab>` 切换至结算核算系统页面。

## 登录后的功能验证流程
进入“结算核算子系统”后，将基于当前的代码修改任务进行自动化或辅助验证：
1. **分析修改内容**：回顾当前开发或修改的代码逻辑（例如：财务核算的日切日期调整、资金结算的过滤逻辑等）。
2. **定位业务菜单**：使用 `browser-use --connect state` 解析结算核算系统的左侧导航栏或主菜单，根据修改内容匹配对应的业务菜单（如“会计核算”、“资金结算”等）。
3. **导航至对应界面**：通过 `browser-use click <index>` 逐级点开菜单树，进入最终的业务操作界面。
4. **辅助测试与验证**：协助用户在界面中输入测试数据、点击查询或提交按钮，观察并反馈系统页面表现，以确认代码修改已正确生效。

可以通过调用 `browser-use` 技能按照上述步骤依次执行。执行到密码输入完毕后，直接通知用户接管。
