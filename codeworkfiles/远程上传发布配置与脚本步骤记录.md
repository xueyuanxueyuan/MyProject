# 远程上传发布配置与脚本步骤记录

## 1. 任务目标
创建一个 SSH 远程上传发布文件的配置文件，通过该文件配置目标服务器信息（地址、端口、服务名称、用户名、密码、默认地址等），并实现一个自动化脚本读取该配置，将项目根目录下 `release` 文件夹中的打包文件一键上传至远程服务器。

## 2. 解决思路与实现步骤

### 2.1 创建通用配置文件
为了方便管理多个地区（如枣庄、临沂等）的服务器信息，我设计并生成了一个 JSON 格式的配置文件，放置在项目的统一管理目录中：
- **配置文件路径**：`/home/source/Jetbrains/Probject/Gjj/config/deploy_servers.json`
- **配置内容设计**：以地区名称作为 Key，配置内容包括 `server_name`（服务名称）、`host`（服务器地址）、`port`（端口）、`username`（用户名）、`password`（密码）、`default_path`（远程默认发布地址）。

### 2.2 编写自动化部署脚本
单纯的配置文件无法自动运行，因此我编写了一个健壮的 Python 脚本，以满足自动化的要求：
- **脚本路径**：`/home/source/Jetbrains/Probject/Gjj/scripts/remote_deploy.py`
- **核心逻辑**：
  1. 通过传入的参数（地区名称，例如 `zaozhuang`）读取并解析 `deploy_servers.json` 中的服务器认证信息。
  2. 使用 `paramiko` 模块进行 SSH 和 SFTP 连接（若环境中未安装 `paramiko`，脚本会自动调用 pip 安装）。
  3. 自动扫描对应地区根目录下的 `release` 文件夹中的所有制品文件（如 `*-app.jar` 和 `.zip` 文件）。
  4. 将文件安全、稳定地通过 SFTP 协议传输到服务器指定的 `default_path`。若远程路径不存在，还会自动尝试创建。

### 2.3 抽象为 AI 技能 (Skill)
为了在以后的交流中，你可以直接通过口令（如“帮我把打好的包上传到远程服务器”）触发该流程，我通过 `skill-creator` 工具创建了名为 `gjj-remote-deploy` 的自动化技能：
- **技能路径**：`.trae/skills/gjj-remote-deploy/SKILL.md`
- **行为规范**：当检测到你的上传请求时，AI 会自动调用 `python3 scripts/remote_deploy.py <地区名称>`，实现真正的“一键发布”。

## 3. 使用方法
1. 在 `config/deploy_servers.json` 中填入你的真实服务器密码和地址。
2. 随后只需对我说：“**把枣庄的包上传到远程服务器**”即可。
3. 也可以在终端手动执行：`python3 scripts/remote_deploy.py zaozhuang`
