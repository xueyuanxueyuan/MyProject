## 核心规则要点提炼

### 架构与分层（必须遵守）
1. **DDD四层架构**: Interface/API -> Application -> Domain -> Infrastructure
2. **依赖方向**: 严格单向，Domain层不依赖基础设施
3. **禁止项**: Controller直调DAO，Domain层出现框架注解

### 命名规范（必须遵守）
1. **拼音缩写 + 驼峰**: 所有标识符使用中文语义的拼音缩写
2. **大驼峰**: 类名、接口名
3. **小驼峰**: 变量名、方法名
4. **后缀规范**: DTO/DO/VO/Entity/Enum后缀必须保留

### 文档约定（必须遵守）
1. **文档目录映射**:
   - `doc/rules/`: 开发规范
   - `doc/prompt/`: 提示词
   - `doc/design/`: 设计文档
   - `doc/demand/`: 需求文档
   - `doc/analysis/`: 需求分析
   - `doc/review/`: 评审与复盘
   - `doc/proto/`: 接口协议
   - `doc/skills/`: 技能库（需同步至 `.cursor/skills/`）

### 智能体协同（必须遵守）
1. **标准工件**: 必须使用 app_spec.txt、feature_list.json、claude-progress.txt
2. **发布检查**: 发布前必须完成 release-checklist.md 逐项勾检
3. **提交规范**: 提交说明固定追加"（AI生成提交）"
4. **差异保留**: 代码变更先保留差异证据再写入目标文件

### 工具链（必须遵守）
1. **vfox管理**: JDK、Node.js、Maven由vfox统一管理
2. **后端要求**: JDK 17.0.2+8
3. **切换命令**: `vfox use java@17.0.2+8`
4. **发布脚本**: 使用项目提供的 remote_deploy.py 和 remote_exec.py
