# SDK检查与切换逻辑 - 步骤与思路

## 概述
本文档记录了在打包的环境配置与验证时，增加SDK检查和切换逻辑的完整步骤和思路。

## 需求说明
在进行打包编译发布前的环境配置与验证时：
1. 首先检查当前环境是否已配置好对应的SDK版本
2. 如果没有找到对应的SDK，则使用 `vfox use` 命令尝试切换
3. 如果 vfox 也没有对应的SDK，则告知用户需要安装对应的SDK

## 涉及的SDK版本
- **后端（Java）**：`java@17.0.2+8`
- **前端（Node.js）**：`nodejs@16.20.2`
- **构建工具（Maven）**：`maven@3.9.14`

## 检查与切换流程

### 流程1：后端SDK检查与切换
```bash
# 步骤1：检查当前Java版本
java -version

# 步骤2：如果版本不符合，使用vfox切换
vfox use java@17.0.2+8

# 步骤3：再次检查Java版本
java -version

# 步骤4：检查Maven版本
mvn -v

# 步骤5：如果Maven版本不符合，使用vfox切换
vfox use maven@3.9.14

# 步骤6：再次检查Maven版本
mvn -v
```

### 流程2：前端SDK检查与切换
```bash
# 步骤1：检查当前Node.js版本
node -v

# 步骤2：如果版本不符合，使用vfox切换
vfox use nodejs@16.20.2

# 步骤3：再次检查Node.js版本
node -v

# 步骤4：检查npm版本
npm -v
```

## 判断标准

### Java版本判断
- 目标版本：`17.0.2+8`
- 检查 `java -version` 输出是否包含 `17.0.2`

### Node.js版本判断
- 目标版本：`v16.20.2`
- 检查 `node -v` 输出是否为 `v16.20.2`

### Maven版本判断
- 目标版本：`3.9.14`
- 检查 `mvn -v` 输出中的 Maven 版本

## 异常处理

### 情况1：当前SDK版本正确
- 操作：无需切换，直接继续打包

### 情况2：当前SDK版本不正确，但vfox中有对应版本
- 操作：执行 `vfox use <sdk>@<version>` 切换
- 验证：切换后再次检查版本

### 情况3：vfox中也没有对应版本
- 操作：告知用户需要安装对应SDK
- 提示信息示例：
  ```
  ⚠️  警告：未找到对应的SDK版本！
  - 需要的Java版本：java@17.0.2+8
  - 当前vfox可用版本：（列出可用版本）
  请先安装对应的SDK版本后再继续！
  ```
