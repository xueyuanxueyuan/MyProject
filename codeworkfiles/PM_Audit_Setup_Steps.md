# 项目经理与审计角色初始化：解决思路与执行步骤

## 1. 任务背景与目标
用户（美女程序员 xy）要求我作为项目经理和审计人员，制定一份明确的工作职责与开发者必须遵守的规范。核心目标是保障代码与设计文档质量、防范返工，并要求开发者以特定表格格式响应审查结果。

## 2. 解决思路
- **职责界定**：明确我在项目中的定位，即主动审计、发现问题、输出表格化的修改要求。
- **流程闭环**：确立“审计出具意见 -> 开发者分析 -> 接受并修改标记 / 拒绝并提供依据记录”的工作流。
- **规范落盘**：将这些规则生成正式的 Markdown 文件，存入规范要求的 `codeworkfiles` 目录及其子目录 `audit_reports` 中。
- **记忆强化**：利用 `manage_core_memory` 工具，将该规范写入核心记忆，确保未来的对话和协作中我与其他智能体都能默认遵循这一准则。

## 3. 执行步骤
1. **目录初始化**：在项目根目录下创建 `codeworkfiles/audit_reports` 文件夹，专门用于存放审计结果和规范文档。
2. **规范文档生成**：编写并输出《项目经理与审计人员工作职责及开发者审查规范》（`PM_Audit_Responsibilities.md`），明确职责、要求以及表格模板。
3. **记录解决思路**：按照 xy 的专属要求（“总是将步骤和解决思路生成markdown文件放在codeworkfiles目录下”），生成本步骤记录文档。
4. **核心记忆更新**：将上述制定的核心开发者规范提取为 Rule 类型的长期记忆，以便持续生效。
