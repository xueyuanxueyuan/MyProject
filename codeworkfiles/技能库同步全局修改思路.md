# 技能库同步全局修改思路

## 1. 目标
按照用户要求，将整个 Gjj 项目内所有创建和引用的技能（Skills）的唯一真值库限定在 `doc/skills` 目录。并且规定智能体在创建和引用技能时，必须将内容同步到自身对应的技能目录中（如 Trae 的 `.trae/skills`，Cursor 的 `.cursor/skills`）。

## 2. 操作步骤

### 2.1 更新 Core Memory 全局规则
调用了 `manage_core_memory` 工具，更新了名为“文档与技能默认规范（Gjj /doc 约定）”的全局规则（Memento ID: 01KMZ1MCZTFHVEABG4G3QTT3M8）。
新增的核心规则内容为：
> - `/doc/skills` 为唯一的主技能库源文件目录。所有技能（Skills）的创建、修改和引用都必须基于该目录进行。
> - 技能必须双向同步至智能体专属目录：如使用 Cursor 则同步至 `.cursor/skills`，使用 Trae 则同步至 `.trae/skills`。

### 2.2 同步现有技能库
将当前 Trae 环境下（`.trae/skills/`）已经存在的技能，全量拷贝并同步至项目的 `doc/skills/` 目录中，确保规范生效时，主技能库不为空且内容保持一致。

### 2.3 后续影响
经过本次修改，以后任何针对 Gjj 仓库的智能体，当它接到“创建技能”或“使用技能”的任务时，都会从其自身记忆中提取这一强制性约束：首要操作点都是 `doc/skills` 目录，最后再拷贝映射到它的运行时目录。
