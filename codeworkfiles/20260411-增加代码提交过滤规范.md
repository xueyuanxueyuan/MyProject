# 项目规范增加：代码提交限制

## 1. 任务背景
用户要求增加项目规范：在提交代码时，只允许提交修改的源码文件，严禁提交任何配置文件（如 `pom.xml`, `application.yml`, `.env` 等），除非用户明确指示。

## 2. 解决思路
为了确保该规则在以后的智能体协作和团队开发中被严格执行，采取了以下操作步骤：

1. **核心记忆持久化**：
   - 使用 `manage_core_memory` 工具，将该规则持久化记录在 `User Scope` 的 `Rule` 类别中，以便智能体在所有项目中都能记住并遵循此规则。
   
2. **项目规范文档更新**：
   - 定位到 `doc/rules/Gjj项目组智能体协同开发规范.md` 文档。
   - 在第 `11. 增量共享规则` 章节中，追加了提交内容过滤的具体要求：“**提交内容过滤**：提交代码时，仅允许提交修改的源码文件，严禁提交任何配置文件（如 `pom.xml`, `application.yml`, `.env` 等），除非用户明确指示需要提交配置文件。”
   
3. **沉淀文档生成**：
   - 遵循用户工作流偏好，在根目录创建 `codeworkfiles` 文件夹（若不存在）。
   - 将本次的解决思路和步骤以 Markdown 形式保存至该目录下，实现留痕。

## 3. 最终效果
- 智能体在未来的操作中，将遵循核心记忆拦截非法的文件提交。
- 团队其他成员或新接入的协作角色也能在项目组协同开发规范文档中清晰地查看到这条约束。
