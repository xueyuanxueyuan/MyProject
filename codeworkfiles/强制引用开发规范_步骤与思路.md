# 强制引用开发规范的解决思路与步骤

## 1. 需求分析
- **目标**：将 `/home/source/Jetbrains/Probject/Gjj/doc/rules` 中的规则强制引用，使不同开发者均遵守该规范。
- **背景**：Gjj 项目组制定了详细的后端开发规范（如 DDD 分层架构、拼音缩写命名法等），但如果缺乏强制约束机制，开发者及其 AI 助手容易在日常开发中忽略这些规则。

## 2. 解决思路
在现代 AI 辅助开发环境（如 Trae / Cursor 等 IDE）中，强制多开发者遵守规范的最佳实践是在项目根目录配置全局规则文件（如 `.cursorrules` / `.traerules`）。
该文件会被提交到代码仓库中，当任何开发者拉取代码并使用 AI IDE 打开项目时，AI 助手会自动读取该文件，并在生成、重构、审查代码时严格执行其中定义的规则。

## 3. 执行步骤
1. **分析现有规范文档**：
   - 读取了 `/doc/rules/Gjj项目组后端开发规范.md` 的内容，提取了核心约束点：DDD 四层架构（接口层 -> 应用层 -> 领域层 -> 基础设施层）、拼音缩写+驼峰命名规范、包目录划分、工具链要求等。
2. **创建全局 AI 规则配置文件**：
   - 在项目根目录创建 `.cursorrules` 和 `.traerules` 文件，以适配不同主流 AI IDE。
   - 在规则文件中明确写入指令，要求 AI 必须强制读取并遵守 `doc/rules/Gjj项目组后端开发规范.md` 中的所有条款。
   - 提炼了部分极度重要的规则（如架构分层、包职责边界、拼音缩写命名）直接写入根目录配置文件中，以提升 AI 遵守的优先级。
3. **落实个人偏好规则**：
   - 检查并确认了 `codeworkfiles` 目录已创建。
   - 将本次任务的解决思路与步骤生成了本 Markdown 文档，存放在该目录下。

## 4. 预期效果
通过在项目根目录引入该规则文件，所有使用该仓库的开发者在借助 AI 进行代码生成、补全或 Code Review 时，AI 均会以 `Gjj项目组后端开发规范` 为最高准则，并在生成代码后自动检查完整性与合适性，从而在团队层面实现了代码规范的强制落地与统一。
