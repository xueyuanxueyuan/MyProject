# 枣庄公积金测试系统验证流程与技能使用指南

## 技能概览：`zzgjj-login`
本技能用于协助开发者在本地 Chrome 浏览器中自动化登录枣庄公积金测试系统 (`http://test.zzgjj:9090/#/`)，并一键直达“结算核算系统”以准备功能验证。

## 核心设计理念
1. **本地浏览器优先**：所有操作均使用用户的本地 Chrome 浏览器 (`browser-use --connect`) 进行，保证会话的可见性和持久性。
2. **人机协同验证码处理**：剥离不稳定的验证码 OCR 环节。AI 负责输入账号密码，用户负责输入验证码，确保 100% 的登录成功率。
3. **领域专注**：登录完成后自动切换进入用户专属的业务领域——“结算核算子系统”。

## 完整的系统验证操作流程

### 阶段一：环境唤起与账号填充 (AI 自动执行)
当用户说出 **“登录枣庄测试系统”** 时：
1. AI 使用 `browser-use` 唤起并连接本地 Chrome 浏览器，打开测试地址 `http://test.zzgjj:9090/#/`。
2. AI 解析页面元素，自动切换至“账号登录”模式。
3. AI 自动填充用户名 `bjzw` 和密码 `123456`。
4. **AI 暂停自动化，并在对话中通知用户**：“账号和密码已输入，请手动输入验证码并点击登录”。

### 阶段二：安全验证与系统切换 (人机协同)
1. **用户**：在弹出的浏览器窗口中，查看图片验证码，手动输入，并点击“登录”按钮。
2. **用户**：在对话中回复 AI：“我已经输入好验证码了，你继续下一步” 或 “已登录”。
3. **AI 接管**：重新使用 `browser-use --connect state` 探测登录后的主工作台。
4. **AI 自动执行**：在综合业务平台中定位到“结算核算子系统”卡片，点击“进入系统”。
5. **AI 自动执行**：切换浏览器焦点至新打开的结算核算系统标签页。

### 阶段三：基于代码变更的业务验证 (AI 辅助验证)
进入结算核算子系统后，正式进入功能验证环节：
1. **需求/代码回顾**：AI 分析当前开发任务或最近修改的代码逻辑（例如：财务核算的日切日期调整、资金结算的过滤逻辑等）。
2. **菜单定位**：AI 解析结算核算系统左侧导航栏的菜单树，结合修改内容匹配对应的功能入口（如 `主页 -> 会计核算 -> ...` 或 `主页 -> 资金结算 -> ...`）。
3. **页面导航**：AI 通过 `browser-use click` 自动点击展开多级菜单，直达最终的业务操作表单或列表页面。
4. **测试执行**：AI 协助用户在页面上填入测试数据（日期、单号等），点击查询或提交。
5. **结果反馈**：AI 提取页面响应结果或弹窗信息，反馈给用户，以确认代码修改是否符合预期。
