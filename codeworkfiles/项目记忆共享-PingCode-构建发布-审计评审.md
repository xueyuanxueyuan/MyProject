# 项目记忆共享

## PingCode

### 项目级记忆

- PingCode 前端 UI 自动化筛选经验：
  - Angular Material 组件的下拉内容通常渲染在 `body` 尾部的 `.cdk-overlay-container` 中。
  - 执行筛选前先点击“重置”，确保面板恢复为仅保留一条默认条件“负责人 属于 当前用户”。
  - 新增筛选条件时，先点击“添加筛选条件”生成第二行，再将字段从“标题”切换为“状态”，然后选择目标状态值，最后点击“确定”。
  - 每一步点击后建议等待 `1000-1500ms`，避免 DOM 尚未完成重渲染导致操作失败。

### 相关用户级补充

- 打开 PingCode 时优先使用本地 Chrome，并使用用户专属项目地址。

## 项目构建打包与远程发布

### 项目级记忆

- Gjj 项目打包编译发布规范：
  - 后端使用 `JDK17(vfox java@17.0.2+8)`。
  - Maven 使用 `3.9.14`，标准构建命令为 `mvn clean package -DskipTests -s settings-tsp.xml`。
  - 需要配置 Aliyun 镜像代理并绕过 HTTP 拦截。
  - 前端使用 `Node 16(vfox nodejs@16.20.2)`。
  - 前端产物打包为 zip。
  - 所有制品统一归档到地区项目根目录下的 `release` 目录。

- Maven 构建镜像配置规则：
  - `settings.xml` 中必须配置 `aliyun-repository` 镜像。
  - `mirrorOf` 需要使用 `!capinfo-tsp-nexus,*`，否则可能导致内部依赖或第三方依赖解析失败。

- 远程发布上传配置与自动化流程：
  - 服务器配置保存在 `config/deploy_servers.json`。
  - 上传脚本使用 `scripts/remote_deploy.py`。
  - 脚本依赖 `paramiko`。
  - 可通过远程发布技能执行 release 制品上传。

## 代码审查与项目审计

### 项目级记忆

- 项目审计与开发者响应规范：
