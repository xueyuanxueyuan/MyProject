# 数据库命名规则修改对比（AI提交）

## 1. 目标
在 `/home/source/Jetbrains/Probject/Gjj/doc/rules/Gjj项目组后端开发规范.md` 中增加明确规定：数据库设计、表命名以及字段命名必须严格遵循“中文拼音缩写 + 驼峰”的格式。

## 2. 修改点及对比

### 修改点一：2.3 命名规范
**原代码：**
```markdown
### 2.3 命名规范（中文拼音缩写 + 驼峰，必须）
- **总原则**：
  - 项目中的变量、对象、接口、方法、类、枚举、包名等，统一遵循**中文语义的拼音缩写命名**。
  - 命名风格统一使用驼峰：类/接口使用**大驼峰**，变量/方法使用**小驼峰**。
```

**新代码：**
```markdown
### 2.3 命名规范（中文拼音缩写 + 驼峰，必须）
- **总原则**：
  - 项目中的数据库表、数据库字段、变量、对象、接口、方法、类、枚举、包名等，统一遵循**中文语义的拼音缩写命名**。
  - 命名风格统一使用驼峰：类/接口/数据库表名使用**大驼峰**，变量/方法/数据库字段名使用**小驼峰**。
...
- **数据库表与字段命名**：
  - 数据库表名：大驼峰 + 拼音缩写，如 `Gdzhxx`、`Jzsq`。
  - 数据库字段名：小驼峰 + 拼音缩写，如 `grzh`、`ywlsh`。
  - （注：全局公共基础字段如 `CREATED_TIME` 等按第 5 节约定保留，业务新增表与字段强制遵循此驼峰规则）。
```

## 3. 解决思路
1. **统一性补充**：虽然之前的后端类名与变量名已经要求了拼音缩写加驼峰，但数据库设计层面容易产生使用下划线（Snake Case）的惯性思维。因此在规范的 `2.3` 节“总原则”中显式地加入了“数据库表、数据库字段”。
2. **前后呼应**：在 `5. 数据访问规范` 章节前置了对数据库的命名格式约束，与前一次加入的“11个公共基础字段”进行了平滑衔接（提示历史基础字段保留其原有大小写规范，但新增的业务字段严格执行驼峰）。
