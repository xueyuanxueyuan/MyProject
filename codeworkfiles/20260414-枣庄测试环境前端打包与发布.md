# 枣庄测试环境前端服务打包与发布记录

## 解决思路
1. 明确目标为“枣庄测试环境”的“前端服务”。
2. 首先利用 `gjj-build-deploy` 规范进行依赖拉取与打包，避免本地环境不一致导致构建失败。
3. 随后利用项目自带的自动化部署工具 `gjj-remote-deploy` 进行构建物的远程分发。
4. 部署后，平滑重启相关的 Kubernetes Deployment 以完成服务的自动更新。

## 操作步骤

1. **确定项目目录并配置环境**：进入枣庄前端项目目录 `prod/WebstormProjects/capinfo-gjj-frontend-jshs-gm`，使用 `vfox` 工具链切换至 Node.js 16。
2. **拉取代码并清理旧数据**：执行 `git pull` 获取最新代码，然后 `rm -rf dist` 清除旧版本内容。
3. **编译打包项目**：
   - 安装项目依赖 `npm install`。
   - 构建项目 `npm run build`。
   - 将 `dist` 目录进行压缩归档：`zip -r capinfo-gjj-frontend-jshs-gm.zip dist`。
4. **归档部署物**：将生成的 `capinfo-gjj-frontend-jshs-gm.zip` 复制并归档到项目专属发布目录 `zaozhuang/release/` 下。
5. **上传远程服务器**：执行 `scripts/remote_deploy.py zaozhuang:test capinfo-gjj-frontend-jshs-gm.zip` 将打包文件自动化上传至枣庄测试环境服务器 (100.70.0.189)。
6. **重启K8s服务**：执行 `scripts/remote_exec.py zaozhuang:test "kubectl rollout restart deployment capinfo-gjj-frontend-jshs-gm-depl -n {namespace}"`，平滑重启前端服务 `capinfo-gjj-frontend-jshs-gm-depl`。
